# Stale policy (대표님 확정)

## 목적
- 업무가 2~3일씩 늘어지는 것을 사람 의지가 아니라 **시스템(상태+핑+질문 1개 강제)**로 방지

## 방치 감지 기준
- doing: **다음 영업일 EOD까지 무업데이트**(야간/주말 오탐 방지)
- blocked: 24h 무업데이트
- triage: 48h 내 최소 산출물(DoD 1줄 + 담당 1명 + 1st 액션 1줄) 없으면 needs-triage

## 알림 단계
1) GitHub 코멘트 + 라벨(needs-info / needs-triage)만
2) (추후) 디스코드 일 1회 요약

## blocked 템플릿(질문 1개 강제)
- Blocker 질문(1개): ____
- 원하는 답 형태: (예/아니오 | 1/2/3 | 링크)

## stale 발생 시 조치
- 상태 자동 전환은 하지 않고 라벨만 추가(needs-info) 후 코멘트로 확인 요청
