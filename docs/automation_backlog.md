# 자동화 백로그

## 목적
- 반복업무를 줄여 실행 속도/정확도 향상
- 장애 인지/복구 시간을 단축하고 운영 표준화를 강화

## 우선순위 Top3 (Issue-B 결과 반영)

### 1) token mismatch 자동 감지/알림
- 대상: `gateway token mismatch`, `device token mismatch`, RPC `unauthorized`
- 기대효과: 수동 점검 의존도 감소, 장애 인지 시간 단축
- 리스크: 오탐/중복 알림 노이즈
- 스코어: 효과 5 / 난이도 3 / 리스크 2

### 2) Gateway 헬스체크 자동화
- 대상: `openclaw gateway status`, `openclaw doctor --non-interactive` 주기 점검
- 기대효과: 장애 조기 발견, 상태 수집/보고 기준화
- 리스크: 점검 주기 과도 시 불필요한 리소스 사용
- 스코어: 효과 4 / 난이도 2 / 리스크 2

### 3) 백업·복구 루틴 자동화
- 대상: 설정/크리덴셜/세션 백업 점검 + 복구 절차 문서 연계
- 기대효과: 복구 표준화, 재현 가능한 대응 확보
- 리스크: 자동화 범위 설정 오류 시 복구 절차 혼선
- 스코어: 효과 4 / 난이도 3 / 리스크 3

## 공통 실행 기준
- 모든 자동화는 dry-run/검증 단계를 우선 적용
- 결과 보고 포맷: `DONE | IN PROGRESS | BLOCKED` 1줄
- BLOCKED 시 원인/필요 지원 1줄 즉시 공유
